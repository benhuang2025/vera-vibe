# 🛡️ Brevis Vera : ZK-Driven Media Authenticity & Traceability

**Brevis Vera** is a prototype system built on Pico ZKVM, designed to combat the "identity crisis" of media in the AIGC era. It allows photographers to prove the authenticity of their work while maintaining the ability to perform legitimate edits (e.g., cropping, brightness adjustment). These edits are verified via Zero-Knowledge Proofs (ZKP), ensuring that the final media is both **traceable** and **authentic** without exposing sensitive raw data.

---

## 📂 Project Structure

- **`brevis-vera-zk/`**: Core workspace containing ZKVM Guest programs, Prover service, and local signing tools.
- **`images/`**: Directory for source media and raw photographs.
- **`web-verifier.html`**: A standalone, premium web verifier for real-time pixel-level hash validation in the browser.
- **`test_e2e.sh`**: A complete end-to-end automated test suite for the entire lifecycle.
- **`spec-v2.md`**: Detailed technical specifications and architectural logic.

## 🚀 Quick Start: Experience Brevis Vera in 3 Steps

### Step 1: Run the End-to-End Test
Execute the automated script in your terminal to simulate the full lifecycle — from capture and signing to ZK proof generation.

```bash
chmod +x test_e2e.sh
./test_e2e.sh
```

**What happens:**
- Generates `private_key_test.pem` (Simulated camera private key).
- Generates `signed_test.json` (Original signed photo metadata).
- Generates `edited_test.png` (Processed image with crop/brightness edits).
- Generates `proof_package_test.json` (The ZK proof package generated by Pico ZKVM).

---

### Step 2: Verify via Web UI
No coding environment required for verification. Use our turnkey web interface:

1.  Open **`web-verifier.html`** in your browser.
2.  Drag and drop **`proof_package_test.json`** into the left JSON zone.
3.  Drag and drop **`edited_test.png`** into the right Image zone.
4.  Click **"Verify Media Authenticity"**.

**Result:** A futuristic 3D scanning animation will reveal the verification status, edit history, and mathematical proof of origin.

---

### Step 3: Test Tamper Detection (Negative Test)
Challenge the system's integrity:

1.  Modify `edited_test.png` using any image editor (even changing a single pixel).
2.  Re-verify it in **`web-verifier.html`**.
3.  **Result:** The verifier will immediately trigger an `❌ Image Hash MISMATCH!`, rejecting the tampered media.

## 🖼️ How to Use Your Own Photos?

Want to test with your own shots? It's simple:
1. Place your photo (JPG or PNG) into the **`images/`** directory.
2. Open **`test_e2e.sh`** and update the `SOURCE_IMAGE` variable on line 22 to point to your filename.
3. Re-run `./test_e2e.sh`.
